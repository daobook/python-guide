风格指南指引¶

与所有文档一样，拥有一致的格式有助于使文档更容易理解。为了使指南更容易被消化，所有的贡献都应酌情符合本风格指南的规则。

指南的写法是 reStructuredText。

注解

指南的部分内容可能还不符合本风格指南。请随时更新这些部分，以便指南的风格指南保持同步

注解

在渲染的 HTML 的任何一页上，你可以点击 “显示源码” 来查看作者是如何设计页面的。

相关性¶

努力使任何贡献与 指南的目的 相关。

  • 避免包括太多与 Python 开发没有直接关系的主题的信息。
  • 如果信息已经存在，更愿意链接到其他来源。一定要描述你所链接的内容和原因。
  • 在需要的地方 引用 参考资料。
  • 如果一个主题与 Python 不直接相关，但与 Python 结合起来很有用（例如，Git、GitHub、数据库），通过链接到有用的资源来参考，并描述为什么它对 Python 有用。
  • 有疑问时，请询问。

标题¶

使用下列风格作为标题。

章节标题：

#########
Chapter 1
#########

章节标题：

*******************
Time is an Illusion
*******************

小节标题：

Lunchtime Doubly So
===================

小节标题：

Very Deep
---------

换行¶

以 78 个字符包住文本行。必要时，行的列数可以超过 78 个字符，特别是如果包裹会使源文本更难以阅读。

使用标准的美式英语，而非英式英语。

序列逗号 （serial comma） （也称为 Oxford comma，牛津逗号）的使用是 100% 没有选择的。任何尝试以缺少的连续逗号提交内容，将导致该项目的永久性移除，因为完全缺乏品味。

流放？ 您在开玩笑吗？ 希望我们永远不必找出来。

代码例子¶

所有代码示例要在 70 个字符进行换行，以避免出现水平滚动条。

命令行例子：

.. code-block:: console

    $ run command --help
    $ ls ..

对于 Unix 控制台的例子，请确保在每一行之前包括 $ 前缀。

对于 Windows 控制台的例子，使用 doscon 或 powershell 代替 console，并省略 $ 前缀。

Python 解释器的例子：

Label the example::

.. code-block:: python

    >>> import this

Python 例子：

Descriptive title::

.. code-block:: python

    def get_answer():
        return 42

外部链接¶

  • 链接，倾向于为众所周知的主题（如专有名词）贴上标签：

    Sphinx_ is used to document Python.

.. _Sphinx: https://www.sphinx-doc.org

  • 倾向于使用带有内联的描述性标签，而不是留下裸露的链接：

    Read the `Sphinx Tutorial <https://www.sphinx-doc.org/en/master/usage/quickstart.html>`_

  • 避免使用诸如 “点击这里”、”这个” 等标签，而是选择描述性标签（具有 SEO 价值）。

链接到《指南》中的各章节¶

要交叉引用本文档的其他部分，请使用 :ref: 关键字和标签。

为了使参考标签更加清晰和独特，总是添加一个 -ref 后缀：

.. _some-section-ref:

Some Section
------------

注意和警告¶

在做笔记时要使用适当的 admonitions 指令。

注意

.. note::
    The Hitchhiker’s Guide to the Galaxy has a few things to say
    on the subject of towels. A towel, it says, is about the most
    massively useful thing an interstellar hitch hiker can have.

警告：

.. warning:: DON'T PANIC

.. todo::
    Learn the Ultimate Answer to the Ultimate Question
